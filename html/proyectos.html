01
E-commerce Full Stack

Juice Shop

Tienda online de jugos naturales

El problema

Un negocio local de jugos necesitaba vender online pero las plataformas existentes cobraban comisiones altas y no se adaptaban a su flujo de trabajo.

Mi rol

Desarrollo completo del proyecto:

  • Diseño de la base de datos
  • Desarrollo backend y frontend
  • Integración de pagos
  • Despliegue y capacitación al cliente

Decisiones técnicas

Django por su panel de administración integrado — redujo semanas de desarrollo sin sacrificar funcionalidad.

  • SQLite en desarrollo para acelerar pruebas; preparado para migrar a MySQL en producción
  • Carrito basado en sesiones — usuarios pueden agregar productos sin registrarse
  • Fusión automática del carrito con la cuenta al autenticarse

Desafíos

El carrito debía funcionar sin registro y conservar productos al crear cuenta

Combinación de datos en sesión con el modelo de usuario al autenticarse

Primera integración con API de pagos externa

Documentación exhaustiva y pruebas en sandbox antes de producción

Panel admin no mostraba pedidos en tiempo real

Sistema de notificaciones para alertar nuevos pedidos sin recargar

Resultados

<2s Carga
1 sem Panel admin
0% Comisiones

Aprendizajes

  • Priorizar soluciones prácticas: usar el admin de Django permitió entregar valor más rápido
  • Buenas prácticas orientadas a escalabilidad y mantenimiento

02
Machine Learning Data Science En desarrollo

Predicción de Partidos

Sistema predictivo de fútbol con XGBoost

El problema

Predecir resultados de partidos de fútbol a partir de datos históricos, reduciendo el impacto del azar y traduciendo el desempeño de los equipos en estadísticos medibles.

El reto: trabajar con datos ruidosos y desbalanceados, donde empates y victorias del no favorito ocurren con menor frecuencia.

Mi rol

Pipeline completo de ciencia de datos (proyecto en desarrollo):

  • Exploración y análisis de datos
  • Limpieza y unificación de múltiples fuentes
  • Feature engineering (+50 variables)
  • Entrenamiento y evaluación de modelos
  • Ajuste de métricas y umbrales de decisión

Decisiones técnicas

XGBoost por su manejo nativo de valores faltantes y soporte para datos desbalanceados mediante scale_pos_weight.

  • Optuna para optimización — más eficiente que GridSearch en espacios amplios
  • Validación cruzada estratificada para mantener proporciones reales
  • Ajuste dinámico del threshold para balancear precisión y recall
  • Ingeniería de +50 features, evaluando aporte individual al modelo

Desafíos

Modelo inicial predecía siempre al favorito (replicaba cuotas de apuestas)

Eliminación de features con data leakage + enfoque en estadísticas de rendimiento

Empates casi inexistentes por desbalance de clases

scale_pos_weight + cost-sensitive learning + ajuste de threshold por clase

Datasets incompletos e inconsistentes

Pipeline de limpieza: unificar tipos, excluir variables sin respaldo histórico

Predicciones alineadas con favorito = sin valor real

Introducción de value betting: solo apostar cuando probabilidad estimada > cuota implícita

Resultados actuales

0.54 F1-score
52+ Features
✓ Balance clases

Aprendizajes

  • La métrica correcta importa más que el accuracy
  • En datos desbalanceados, optimizar F1 cambia completamente el enfoque
  • Value betting > predecir al favorito siempre

03
Desktop App C# / .NET

Sistema Punto de Venta

Aplicación de escritorio para gestión de ventas

El problema

Pequeños negocios usaban Excel para llevar ventas e inventario — perdían datos y no había control de acceso por empleado.

Mi rol

Arquitectura, desarrollo y testing. Aprendí C# durante el proyecto. Proyecto individual.

Decisiones técnicas

WinUI 3 para UI moderna y nativa en Windows + patrón MVVM para separar lógica de interfaz.

  • SQL Server LocalDB — no requiere servidor externo, ideal para negocios pequeños
  • QuestPDF para generar tickets y reportes profesionales
  • Sistema de roles: Admin, Vendedor, Cajero con permisos diferentes

Desafíos

Primer proyecto en C# — curva de aprendizaje fuerte con async/await y data binding

Documenté todo el proceso y aprendí lo necesario para cada feature, no todo a la vez

Reportes PDF con formato profesional

QuestPDF con templates reutilizables para tickets y reportes

Resultados

3 Roles
<1s Gen. PDF
0 Datos perdidos

Aprendizajes

  • Aprendí un lenguaje nuevo bajo presión — la clave fue no intentar aprenderlo todo
  • MVVM facilita mucho el testing y mantenimiento a largo plazo

04
Data Science Analytics

Análisis Predictivo - Superstore dataset

Pipeline de análisis y modelado predictivo

El problema

Dataset retail con 9,800+ transacciones sin procesar, formatos inconsistentes y ausencia de variables críticas como precio unitario.

Mi rol

Desarrollo completo como Data Scientist: limpieza de datos, feature engineering avanzado, modelado predictivo, evaluación y generación de insights accionables.

Decisiones técnicas

Pipeline reproducible con Pandas — cada paso documentado para que cualquiera pueda replicar el análisis.

  • Limpieza y estandarización de 31 variables base
  • Feature engineering avanzado : +34 variables mediante agregaciones históricas
  • Se eligió Linear Regression como modelo principal tras observar que, con un feature engineering robusto, superó a modelos más complejos

Desafíos

Datos con formatos inconsistentes

Pipeline de limpieza reproducible documentado paso a paso

Modelo inicial con R² ≈ 11% por falta de información

Feature engineering enfocado en reconstruir información faltante

Mejora de +793% en capacidad predictiva

Resultados

95% R²
45 Features finales
+793% capacidad predictiva

Aprendizajes

  • Feature engineering bien dirigido puede superar modelos complejos
  • Un buen gráfico comunica insights que una tabla de números no puede
